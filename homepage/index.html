推出全新评估工具：深入理解 AI 带来的学习成效

OpenAI · 全球事务 · 2026 年 3 月 4 日

教育是 AI 最具潜力的应用前沿之一。借助 ChatGPT 等工具，个性化的学习支持已变得触手可及，让学生能随时随地获得帮助。然而，教育界对 AI 如何影响学习成效的认知仍处于起步阶段。

为了填补这一空白，我们与爱沙尼亚塔尔图大学以及斯坦福 SCALE 项目联合开发了学习成效衡量套件——一套支持跨教育场景进行纵向跟踪、衡量学习成效的系统性框架。

核心要点

  • 当前研究的局限性：现有的 AI 学习影响评估方法仍无法完整刻画 AI 随时间推移对学习成效产生的全周期影响。
  • 标准化衡量框架："学习成效衡量套件"首次为纵向研究提供标准化框架。
  • 推进研究生态建设：OpenAI 的"学习实验室"将联合合作伙伴持续发布研究成果。

"这项研究让我们能够快速获取经验，同时也为深入理解如何将 AI 审慎地整合进学校教育奠定了基础。我们希望了解这些工具在支持严谨学科学习的同时，如何培养高阶思维、创造力、好奇心，以及学习者的自信心。"

— Susanna Loeb，斯坦福大学教育学教授、SCALE 项目主任

起源与早期研究

学生利用 AI 工具进行学习的方式多种多样：既包括直接索取答案，也包括在类似导师的引导下逐步推导问题。为了帮助用户以促进"深度理解"和"技能构建"的方式使用 ChatGPT，OpenAI 推出了学习模式——由教学法专家编写的系统指令驱动，核心行为包括脚手架式引导、理解程度检查与启发式练习。

实验设计

针对 300 多名大学生进行随机研究：对照组使用传统在线资源（如 Google 搜索和 YouTube），另外两个实验组分别使用两种版本的"学习模式"。

研究发现

在微观经济学考试中，被分配使用"学习模式"的学生与对照组相比，成绩显著提升——得分高出约 15%。

虽然这些数据反映了现实世界的差异，但也暴露出传统学习成效衡量方式的深层局限——单次研究无法捕捉 AI 与学习者策略、偏好和习惯共同演化的机制。

构建更完善的衡量体系

基于"学习模式"的研究成果，我们正在构建一套结构化的衡量体系，立足于三大核心信号：模型行为、学习者反馈、以及随时间推移产生的可衡量认知产出。

  • 系统指令：利用自然语言改变模型默认行为，使其与教学法精准对齐。
  • 学习交互分类器：自动检测真实交互中的"学习瞬间"，标注参与度与纠错表现。
  • 学习质量评分器：评估每一个学习瞬间，判断学习者是否达成目标。
  • 纵向学习评分器：在个人和群组层面长期追踪参与度、持久性与元认知策略。
  • 标准化认知衡量：通过接入前 / 中 / 后的第三方量表衡量批判性思维、创造力与记忆力。

未来展望

在广泛开放学习成效衡量套件之前，我们正与塔尔图大学及斯坦福 SCALE 项目合作，在爱沙尼亚开展大规模验证——研究涵盖近 2 万名 16 至 18 岁学生，跨度长达数月。

"爱沙尼亚始终将教育视为一个持续改进的动态系统。随着 AI 融入这一版图，核心挑战在于我们如何衡量 AI 对学习的长期影响。"

— Jaan Aru，塔尔图大学

Agent 使用搭档

Flowix 是 AI Agent 的记忆库与协作层——文档即任务、笔记即上下文，外部工具通过 CLI 与 MCP 直接对接。

可作为 AI Agent 的记忆库

Markdown + YAML 纯文本存储，Agent 直接读写无需转换。

文档即任务 / 审计 / 调度

笔记可作为任务清单、审计日志、调度凭据，供 Agent 调用。

CLI & MCP 支持

内置 CLI sidecar + MCP server，外部 Agent 与工具无缝对接。

数据本地保存，离线使用

全部笔记以 .md 文档保存在电脑中（或私有云盘），仅大模型联网使用。

笔记是硬盘上的 Markdown + YAML 文件，纯文本——Vim、VS Code、Obsidian，任何编辑器都能直接打开。换台机器，整个数据目录打包就能带走。

没有云端同步、没有遥测、没有埋点。AI 在本地读你的文件、写你的文件——只有你主动按下的那一句 prompt 才会离开本机。配置存在本地 JSON，源码以 CC BY-NC 4.0 协议开源，每一行都能读。

离线也能写、也能读——网络只在让 AI 干活时才需要。

适合谁 · 不适合谁

适合你，如果…

  • 想让 AI 接着你的笔记写，而不是从零开始
  • 在 Markdown 里写 prompt、需求、规范、工作流
  • 用笔记启动 Agent、Codex、Claude 让它们接着干
  • 重视数据主权，文件能直接打开

不适合你，如果你需要…

  • 只需要简单记事（系统便签就够了）
  • 团队协作 / 多人同步
  • 双链 / 知识图谱
  • 离不开插件生态
