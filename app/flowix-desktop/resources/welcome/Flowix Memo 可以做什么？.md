# Flowix Memo 可以做什么？

Flowix Memo 是一款本地优先的笔记应用。它把笔记当作“工作素材”，而不是单纯的“存放仓库”。你写下的内容既可以被自己反复回看，也可以交给 AI 代理加工、总结、改写和延展。

## 长期沉淀

Flowix Memo 的笔记以本地 Markdown 文件存在。你可以直接备份这些文件，也可以把它们放进自己的同步盘。内容不被锁在某个封闭系统里，这一点适合长期积累资料、想法、项目记录和个人知识库。

## 按项目分流

Flowix 用笔记本隔离不同上下文。一个笔记本就是一个文件夹，你可以为客户、研究方向、个人记录和工作内容分别建立笔记本。需要切换时，打开状态栏的笔记本切换器即可。

这种方式的好处是边界清楚。你不会把所有内容都塞进同一个空间，也不需要为每个主题维护复杂层级。一个项目、一个领域或一个长期用途，都可以对应一个笔记本。

## AI 协作

Flowix 可以在文档里直接调用内置 AI 代理，也可以连接本地 CLI 代理，例如 Claude Code、Codex 和 Hermes。你可以让 agent 阅读笔记、生成摘要、改写内容、整理行动清单，或者基于项目上下文完成更复杂的工作。

对新手来说，最自然的用法是先把内容写进笔记，再让 AI 代理帮你加工。这样 agent 的输入来自你的真实上下文，输出也能留在同一篇笔记或同一个笔记本里，后续回看更方便。

## 结构化与轻量化并存

Flowix 支持标签和 frontmatter 属性，但不会要求你一开始就建立复杂系统。普通写作可以直接当 Markdown 文本随手记；当某篇笔记需要长期维护、筛选或检索时，再加标签和属性。

这种方式适合从轻量开始。你可以先记录内容，等内容变多、用途变明确之后，再逐步补充结构。

## 不适合实时协作编辑

Flowix 不是多人实时协同工具。它更适合一个人长期积累内容，再通过 AI 协作提高整理和产出效率。如果你的核心需求是多人同时编辑同一份文档，专门的在线协作文档工具会更合适。

## 不适合重型排版

Flowix 的核心是 Markdown，不是复杂富文本排版，也不是 block-based 页面编辑器。它更重视内容的可读、可写、可迁移，而不是制作复杂页面。

如果你需要精细排版、复杂媒体布局或高度视觉化的页面，Flowix 可以保存素材和草稿，但不一定是最终发布工具。

## 不适合云端全托管

Flowix 默认使用本地存储。它不会把所有内容自动交给云端服务托管。如果你需要跨设备同步，可以结合自己的同步方案，例如系统同步盘或第三方文件同步工具。

这也意味着你对文件有更直接的控制权。备份、迁移和同步由你决定，而不是由某个固定平台决定。

## 与一般笔记应用的差异

Flowix 的笔记本就是文件夹，你能在系统文件管理器里直接看到 `.md` 文件。标签写在正文里，不需要在侧边栏手动维护分类。笔记属性写在 frontmatter 中，会跟随文件一起移动，也能被其他 Markdown 工具读取。

更重要的是，AI 代理可以围绕你的笔记工作。它不是把笔记当作静态存档，而是把内容变成可以继续加工的上下文。
