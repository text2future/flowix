# 功能介绍：如何快速上手

这一篇带你完成 Flowix Memo 里最常用的四件事：创建笔记本、使用标签、设置属性，以及在阅读视图里回看内容。先掌握这些操作，就能把 Flowix 当作稳定的日常笔记工具使用。

## 创建你的第一个笔记本

笔记本就是一个本地文件夹。你可以在状态栏右下角打开笔记本切换器，新建一个目录，或者注册一个已经存在的目录。确认路径后，Flowix 会把这个目录作为笔记本，并为里面的 Markdown 文件建立索引。

建议给笔记本起一个清晰的名字，例如 `研究`、`工作`、`日记`。名字越贴近日常使用场景，之后切换上下文时越不容易混乱。

完成后，你可以直接在系统文件管理器里看到这些 `.md` 文件。它们不是封闭数据库里的内容，而是真实存在于本地目录中的文件，因此可以备份，也可以放进你自己的同步盘。

## 用标签管理笔记

标签直接写在正文里，例如 `#idea`、`#meeting`、`#todo`。Flowix 会自动识别这些标签，并把它们显示在侧边栏的标签过滤器里。

好标签描述的是你将来怎么找回这条笔记。比如 `#decision` 适合标记最终决策，`#draft` 适合标记还在修改的草稿，`#reference` 适合标记以后可能引用的资料。`#idea`、`#meeting`、`#todo` 则适合按用途快速分类。

普通写作不需要每条都打标签。只有当一篇笔记进入“以后还会回头找”的状态时，再加标签最合适。这样标签系统会保持轻量，也不会变成新的整理负担。

## 设置笔记属性

当标签不够表达结构化信息时，可以使用属性。属性显示在文档右上角的属性面板里，适合记录状态、类型、来源链接和搜索关键词。

例如，Status 可以表示 Draft、In Progress 或 Done；Type 可以表示 Note、Prompt 或 Todo；Source 可以保存引用链接；Keywords 可以补充一些正文里没有直接出现、但你希望以后能搜到的词。

这些属性会写入 Markdown 文件的 frontmatter。也就是说，属性会跟随文件一起移动、备份和同步；任何兼容 frontmatter 的工具，也都能读取这些信息。

## 专注阅读

当笔记进入回看阶段时，建议切到阅读视图，而不是继续停留在编辑视图。阅读视图能减少干扰，让你先完整理解内容，再决定是否需要补标签、加属性，或者交给 AI 代理生成后续动作。

阅读视图特别适合重新过一遍之前的研究资料，也适合在做决策前梳理上下文。如果一篇笔记很长，你也可以先在阅读视图中确认重点，再让 AI 代理把它浓缩成清单或行动项。
