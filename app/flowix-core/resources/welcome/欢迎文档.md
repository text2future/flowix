# 欢迎文档

&nbsp;

Flowix Memo 是 **++AI Agents 的文档空间++**。它常用于解决使用 AI Agent 时遇到的以下问题：

- [ ] Prompt 指令散乱在各处，缺乏管理
- [ ] AI Agent 总是在某些地方踩坑犯错，记忆断片
- [ ] 通过对话跑通的工作流，不知道该怎么沉淀

Flowix Memo 是为 Agent 管理文档与上下文的工具。它可以让 AI 总结工作流、常见问题、个人偏好等内容，并沉淀下来反复使用。

同时，个人日常记录也可以成为高质量的输入信息。

---

## 使用场景

### 1. 日常笔记和 AI 需求/记忆/产物管理

可以写读书笔记，旅行路书，工作日报、日常随笔等，满足笔记产品的全部功能。

可把需求、资料、参考链接写进文档，让 Agent 基于内容进行总结、改写、问答，也可以进一步拆解任务、编写代码等。

Agent 的输出也可写回到文档里，形成可以复查、继续编辑和再次调用的记忆。这让 AI 协作从一次性聊天变成可积累的工作流。

### 2. 本地优先，内容可长期沉淀

Flowix Memo 的笔记以本地 Markdown 文件保存。你能在系统文件夹里直接看到这些 `.md` 文件，也可以使用自己的同步盘、备份工具或版本管理方式保存它们。

这种方式不会把内容锁在某个专有云服务里。以后即使更换工具，笔记仍然是普通 Markdown 文件，可以继续被其他编辑器读取。

### 3. 按项目管理上下文

Flowix Memo 用笔记本组织内容。一个笔记本就是一个本地文件夹，你可以把工作、研究、客户项目、日记、资料库或代码项目分别放在不同笔记本里。

当你切换笔记本时，Flowix 会切换当前上下文。这样在处理某个项目时，不会被其他项目的内容干扰；调用 AI Agent 时，也更容易只提供相关资料。

### 4. 让 AI 行为可以被记录和复用

Flowix Memo 支持在文档里调用内置 AI Agent，也可以连接 Claude Code、Codex、Hermes 等本地 CLI 代理。

你可以控制 Agent 能看到什么：当前笔记、某个文件夹、整个笔记本，或一个项目目录。上下文越明确，Agent 的输出越稳定；过程被记录下来后，也更容易复盘、修改和继续推进。

### 5. 通过 Flowix CLI 和 MCP 接入更多 Agent 工作流

Flowix CLI 适合给本地 Agent 做非交互式笔记操作（==可能被 Agent 的沙箱拦截==）。

MCP 则让 Flowix Memo 可以作为标准化工具接入支持 MCP 的其他 AI 客户端。外部 AI Agent 客户端能通过工具调用读取、检索和更新你的文档，让笔记成为长期可用的上下文来源。相比 CLI，MCP 更推荐用于接入外部 AI Agent 客户端。为最大化节约上下文，Flowix Memo 的 MCP 被设计为只提供 1 个工具。

## 快速上手

### 1. 创建或注册笔记本

笔记本就是一个本地文件夹。新建空文件夹，或注册已有文件夹，即可让 Flowix Memo 管理其中的 Markdown 文件。

### 2. 写下可供 Agent 使用的上下文

一篇 Flowix 文档既可以是笔记，也可以是 Agent 的任务说明。建议写清楚：

- 背景：要处理什么事。
- 资料：需要参考哪些内容。
- 目标：希望产出什么结果。
- 约束：格式、边界和注意事项。

写得越清楚，Agent 越容易直接接手任务。

### 3. 用标签管理笔记

标签直接写在正文里，例如 `#idea`、`#meeting`、`#todo`、`#prompt`、`#agent`。Flowix Memo 会自动识别，并显示在侧边栏过滤器中。

### 4. 用属性补充结构化信息

当标签不够用时，可以用属性记录更稳定的信息，例如 `status`、`kind`、`source`、`agentRole`、`keywords`。
